### Auto-Deploy Workflow
1. Push code to GitHub `main` branch
2. Cloudflare detects commit automatically
3. Runs `npm install` (installs dependencies)
4. Runs `npm run build` (builds project)
5. Deploys `dist/` to edge network globally
6. Site live at https://crs-web-1.pages.dev

## ðŸŽ“ LESSONS LEARNED

### Technical Insights
1. **Cloudflare Workers requires serveStatic** - Static files need explicit middleware
2. **Tailwind v4 is different** - Requires @tailwindcss/postcss, not tailwindcss plugin
3. **CSS must be in build process** - Can't rely on public/ folder for Hono/Workers
4. **Light mode > Dark mode** - Better accessibility and readability for most users
5. **Physics-based design** - Color systems with rationale are more maintainable

### Design Insights
1. **Invisible elements are deal-breakers** - Logo visibility is critical
2. **ALL CAPS reduces readability** - Use sparingly for headings only
3. **Contrast matters** - WCAG isn't just compliance, it's usability
4. **Light backgrounds feel more spacious** - Dark mode felt claustrophobic
5. **Brewforce works** - Physics-based color system is unique and memorable

### Process Insights
1. **Git commits save lives** - 33 commits allowed easy rollback when needed
2. **Documentation is essential** - 8 comprehensive docs made troubleshooting easy
3. **Test locally before deploy** - Caught issues that would've failed in production
4. **Auto-deploy is powerful** - Push to main = live in 5 minutes
5. **GitHub authentication needed** - Setup upfront saved time later
